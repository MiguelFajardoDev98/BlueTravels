INTRODUCCIÓN

Blue Travels es una plataforma de servicios turísticos desarrollada utilizando el framework Angular, diseñada para ofrecer a los usuarios una experiencia interactiva para explorar, filtrar y reservar experiencias turísticas relacionadas con el avistamiento de ballenas y actividades náuticas en diversas ubicaciones de América Latina.

Este proyecto representa la aplicación práctica de los conceptos aprendidos en el curso de desarrollo frontend con Angular, implementando una arquitectura basada en componentes, servicios inyectables y manejo de estado mediante RxJS y localStorage para la persistencia de datos del lado del cliente.

Durante el desarrollo, se han implementado características clave como:
- Navegación entre vistas mediante el enrutador de Angular
- Catálogo dinámico de tours utilizando *ngFor
- Sistema de filtrado de tours por destino y tipo de experiencia
- Persistencia de datos utilizando localStorage
- Formularios interactivos para crear y editar tours
- Diseño responsive adaptable a diferentes tamaños de pantalla

El proyecto ha sido desplegado exitosamente en GitHub Pages, permitiendo su acceso público mediante el enlace: https://miguelfajardodev98.github.io/BlueTravels/

FUNCIONALIDADES TÉCNICAS

Uso de *ngFor para el Catálogo Dinámico

En el componente CatalogoComponent, se utiliza la directiva *ngFor de Angular para renderizar dinámicamente la lista de tours filtrados:

<div class="grid-3">
    <article *ngFor="let servicio of serviciosFiltrados" class="service-card">
        <!-- Contenido de la tarjeta de servicio -->
    </article>
</div>

Esta directiva itera sobre el observable serviciosFiltrados$ (que emite un array de servicios filtrados) y crea un elemento <article> para cada servicio en el array. Angular actualiza automáticamente la vista cuando el array cambia, lo que permite una actualización eficiente del DOM cuando se agregan, eliminan o modifican tours.

El uso de *ngFor con el pipe async (*ngFor="let servicio of serviciosFiltrados$ | async") permite que Angular se suscriba automáticamente al observable y se desuscriba cuando el componente se destruya, evitando fugas de memoria.

Sistema de Filtrado de Tours

El sistema de filtrado está implementado en el CatalogoComponent mediante una función de filtrado personalizada que combina los datos del servicio con los parámetros de ruta:

this.filteredServicios$ = combineLatest([
  this.servicioService.servicios$,
  this.route.queryParams
]).pipe(
  map(([servicios, params]) => this.filtrarServicios(servicios, params['destino'], params['experiencia']))
);

La función filtrarServicios implementa la lógica de filtrado:

private filtrarServicios(servicios: Servicio[], destino?: string, experiencia?: string): Servicio[] {
  return servicios.filter(s => {
    let matchDestino = true;
    let matchExperiencia = true;

    if (destino) {
      matchDestino = s.ubicacion.toLowerCase().includes(destino.toLowerCase());
    }

    if (experiencia) {
      const keywords: {[key: string]: string[]} = {
        'avistamiento': ['avistamiento', 'encuentro'],
        'fotografico': ['fotográfica', 'foto', 'capturar'],
        'navegacion': ['navegación', 'recorrido', 'barco']
      };
      const searchTerms = keywords[experiencia] || [experiencia];
      matchExperiencia = searchTerms.some(term => 
        s.nombre.toLowerCase().includes(term.toLowerCase()) || 
        s.descripcion.toLowerCase().includes(term.toLowerCase())
      );
    }

    return matchDestino && matchExperiencia;
  });
}

Este sistema permite:
- Filtrado por destino (ubicación) mediante coincidencia parcial insensible a mayúsculas/minúsculas
- Filtrado por tipo de experiencia usando palabras clave predefinidas
- Combinación de ambos filtros simultáneamente
- Actualización en tiempo real al cambiar los parámetros de consulta en la URL

Persistencia con localStorage

La persistencia de datos se implementa en el ServicioService utilizando la API de localStorage del navegador:

private readonly STORAGE_KEY = 'bluetravels_servicios';

private cargarServiciosDesdeStorage(): Servicio[] {
  const stored = localStorage.getItem(this.STORAGE_KEY);
  return stored ? JSON.parse(stored) : this.serviciosDefault;
}

private guardarServicios(servicios: Servicio[]): void {
  localStorage.setItem(this.STORAGE_KEY, JSON.stringify(servicios));
  this.serviciosSubject.next(servicios);
}

Características de la implementación de persistencia:

1. Inicialización: Al crear el servicio, verifica si existen datos en localStorage. Si no existen, inicializa con datos predeterminados
2. Lectura: Los datos se leen desde localStorage y se convierten de JSON a objetos JavaScript
3. Escritura: Cada vez que se modifican los datos (crear, actualizar, eliminar), se guardan de nuevo en localStorage
4. Reactividad: Se utiliza un BehaviorSubject para notificar a los componentes cuando los datos cambian
5. Claves múltiples: Se utilizan diferentes claves en localStorage para diferentes tipos de datos:
   - bluetravels_servicios: Almacena los tours/tours
   - bluetravels_favoritos: Almacena los IDs de los tours marcados como favoritos
   - bluetravels_contactos: Almacena los mensajes de contacto enviados

Esta enfoque garantiza que los datos persistan entre recargas de página y reinicios del navegador, proporcionando una experiencia de usuario más robusta sin requerir un backend.

CONCLUSIONES

Aprendizajes en Angular

Durante el desarrollo del proyecto Blue Travels, se adquirieron y consolidaron los siguientes conocimientos sobre Angular:

1. Arquitectura basada en componentes: Comprensión profunda de cómo estructurar una aplicación Angular utilizando componentes reutilizables y encapsulados
2. Enrutamiento: Implementación efectiva del enrutamiento de Angular para navegación entre vistas sin recarga de página
3. Manejo de estado: Uso efectivo de RxJS (BehaviorSubject, Observable, combineLatest) para manejo reactivo de estado
4. Forms: Implementación de formularios template-driven con validación y manejo de eventos
5. Directivas: Uso efectivo de directivas estructurales (*ngFor, *ngIf) y de atributos ([routerLink], [src], [class])
6. Inyección de dependencias: Implementación correcta de servicios inyectables con providedIn: 'root'
7. Buenas prácticas: Aplicación de convenciones de nomenclatura, separación de responsabilidades y organización de código

Experiencia con Despliegue

El despliegue de la aplicación en GitHub Pages proporcionó valiosos aprendizajes:

1. Configuración de base-href: Importancia de configurar correctamente el base-href para que los recursos se carguen correctamente en entornos de subdirectorios
2. Herramientas de despliegue: Utilización efectiva de angular-cli-ghpages para automatizar el proceso de despliegue
3. Ramas de GitHub Pages: Comprensión de cómo GitHub Pages utiliza ramas específicas (gh-pages) para servir contenido estático

Basado en la experiencia de desarrollo, se identifican las siguientes áreas para mejoras futuras:

3. Optimización de rendimiento: Implementar lazy loading de módulos y estratégicas de cambio de detección para mejorar el rendimiento
4. Testing: Añadir pruebas unitarias y de extremo a extremo utilizando Jasmine y Protractor/Cypress

6. Internacionalización (i18n): Agregar soporte para múltiples idiomas
7. Mejoras de UI/UX: Implementar animaciones, transiciones y microinteracciones para mejorar la experiencia de usuario
